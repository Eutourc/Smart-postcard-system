# Smart Postcard V1.0 Architecture

## 四层分离

### 1. Software
00～15 各软件模块。它们是工程代码与界面逻辑。

### 2. Assets
系统自带资源，包括背景、邮票、官方章、图标、字体、广告素材。
系统资源可放在各模块 `assets/` 或全局 `00_Core/assets/`。

### 3. Database
结构化业务数据，包括注册、订单、纪念事件、公司计数器、推广统计。

### 4. Storage
用户上传的海量照片、视频、录音、电子相册文件。
未来来自全球客户的海量文件不能放 GitHub，应由正式网站的文件存储系统管理。

## 关键数据规则

- 03_Map 是纪念事件主数据源。
- 04_Note 读取 03_Map 的纪念附言数据。
- 09_Postmark 读取 03_Map 的日期数据。
- 11_Payment 只显示支付结果，不显示佣金。
- 12_Register 的公司后台负责计数器与佣金计算。

## GitHub 定位

GitHub 只负责开发、版本管理、备份与协作。
正式运行网站不依赖 GitHub 专有路径。
